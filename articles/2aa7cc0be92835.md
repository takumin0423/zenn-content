## 第2章
### JavaScript
- Mocha -> LiveScript -> JavaScriptと名前が変わっていった
- JavaScriptのことを「Cの皮を被ったLisp」だと言う人がいる
- ECMAScriptはEcma Internationalという業界団体が定めている
- JavaScriptはオブジェクト指向ではあるが、クラスベースではなくプロトタイプベース

### データ型
- JavaScriptのデータ型には「プリミティブ型」と「オブジェクト型」がある
  - プリミティブ型とは、インスタンスメソッドを持たない、オブジェクトではないデータのこと

- falsyなのはfalse, 0, NaN, '', null, undefined
  - それ以外は全部truthy

- nullとundefinedを除くすべてのプリミティブ型に「ラッパーオブジェクト」が存在している
  - プリミティブ型の値に対してアクセスすると、対応するラッパーオブジェクトに自動変換される

- JSONの正式名称は「JavaScript Object Notation」

### 関数
- 文は何らかの手続きを処理系に命令するもの、式は評価された後に値として存在するもの
  - 要するに変数に代入できるのが式、できないのが文

- 文の末尾にはセミコロンが必要だが、`{}`のブロックで終わる場合にはセミコロンはつけない
- 関数宣言文ではなく関数式による定義が推奨されている

```js
// BAD
function double(n) {
  return n * 2;
}

// GOOD
const twice = function(n) {
  return n * 2;
};
```

- 関数は組み込みオブジェクト`Function`のインスタンスであり第1級オブジェクト
  - 第1級オブジェクトとは、変数へ代入したり他の関数の引数として渡したりできるということ

## 第3章
### 関数型プログラミング
- 同じ入力に対して同じ作用と出力が保証されていることを参照透過性と言う
- 式を組み合わせてプログラムを構成する

### コレクション
- JavaScriptの配列の破壊的メソッドを非破壊的に使うには`slice`メソッドを使って配列全体のシャローコピーを作成する
- ES2016から提供されている`includes`メソッドというものがある
  - 指定した値の要素が一つでも含まれているかどうかを真偽値で返す

### カリー化
- 複数の引数を取る関数を、より少ない引数を取る関数に分割して入れ子にすること
- カリー化された関数の一部の引数を固定して新しい関数を作ることを関数の部分適用と呼ぶ

### クロージャ
- 関数と、その関数が作られた環境を一つにした特別なオブジェクト
- 高階関数が外の関数の引数を内部で参照したり、関数の部分適用で引数を固定したりできるのはクロージャという仕組みのおかげ
- Reactの関数コンポーネントはクロージャであることが多い

### Promise
- ES2015から追加された、非同期処理を行うためのJavaScript標準組み込みオブジェクト
- finallyを使えるのはES2018から
- TypeScriptだと一番上の階層でいきなりawaitを使うことができる機能がバージョン3.8から追加された
